Question: 1

What is the difference between .map(), .forEach(), .filter(), and .find()?

Answer :

There are many key difference between .map(), .forEach(), .filter(), and .find(). Every developer should know about this in order to do his work perfectly.


Differences :


  1. .map() executes the same code on every element in an array and returns a new array with the updated elements.

  2. .forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.

  3. .filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.

  4. .find() is also a search function like the previous but they differ in one small detail — this function returns only one match in an array. If in an array is more than one result, the function will return the first that has matched.

What is the difference between Local Storage, Section Storage, Cookies?

Answer:

Differences are given bellow:


  • Storage Limit

    • Local Storage stores up to 10 megabytes and Session storage stores up to 5 megabytes. On the other hand, these provide a very restrictive and small storage capacity of 4 kilobytes for Cookies.


  • Accessibility

    • If we talk about it, since session storage is tied to the particular session and each tab has its session, data is only available in the current tab in which we’ve set the session storage data. Lastly, cookies are somewhat similar to local storage as they are accessible from any window or tab. It could also be accessed on the server


  • Expiration

    • Session storage stores data only for a session, meaning that the data is stored until the browser (or tab) is closed, for Local Storage it have to delete manually and for cookies can be updated , set using document.cookie object from browser window object.

    What is event loop? How does it work?

    Answer :

    Definition:
    The event loop is the secret behind JavaScript’s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures,it gives us the illusion of multi-threading


    Working method:

    The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process.
