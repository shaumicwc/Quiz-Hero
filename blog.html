Question: 1

What is the difference between .map(), .forEach(), .filter(), and .find()?

Answer :

There are many key difference between .map(), .forEach(), .filter(), and .find(). Every developer should know about this in order to do his work perfectly.


Differences :


  1. .map() executes the same code on every element in an array and returns a new array with the updated elements.

  2. .forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.

  3. .filter() checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.

  4. .find() is also a search function like the previous but they differ in one small detail — this function returns only one match in an array. If in an array is more than one result, the function will return the first that has matched.

Question: 3

What is event bubble?

Answer :

Definition:
When an element receives an event, and that event is transmitted to its parent elements in the DOM tree until it gets to the root element. This concept is called event bubble.


Question: 4

What is a callback function and why will you use it?

Answer:

When a function is sent as an argument then it executes after the main function has been executed it’s called a Callback Function. The callback function is very useful for its efficiency. Callback Functions give a way to you to set up your program in such a way that events will trigger the correct code. It’s very common in programs with GUIs where users can click on any element on the UI and the appropriate code will be run.
